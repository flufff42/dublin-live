439=head2 execute_tests( tests => \@test_files, out => \*FH )
440
441Runs all the given C<@test_files> (just like C<runtests()>) but
442doesn't generate the final report. During testing, progress
443information will be written to the currently selected output
444filehandle (usually C<STDOUT>), or to the filehandle given by the
445C<out> parameter. The I<out> is optional.
446
447Returns a list of two values, C<$total> and C<$failed>, describing the
448results. C<$total> is a hash ref summary of all the tests run. Its
449keys and values are this:
450
451 bonus Number of individual todo tests unexpectedly passed
452 max Number of individual tests ran
453 ok Number of individual tests passed
454 sub_skipped Number of individual tests skipped
455 todo Number of individual todo tests
456
457 files Number of test files ran
458 good Number of test files passed
459 bad Number of test files failed
460 tests Number of test files originally given
461 skipped Number of test files skipped
462
463If C<< $total->{bad} == 0 >> and C<< $total->{max} > 0 >>, you've
464got a successful test.
465
466C<$failed> is a hash ref of all the test scripts that failed. Each key
467is the name of a test script, each value is another hash representing
468how that script failed. Its keys are these:
469
470 name Name of the test which failed
471 estat Script's exit value
472 wstat Script's wait status
473 max Number of individual tests
474 failed Number which failed
475 canon List of tests which failed (as string).
476
477C<$failed> should be empty if everything passed.
